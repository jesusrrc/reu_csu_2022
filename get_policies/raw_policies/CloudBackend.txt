<h2>Distributed Data Management</h2>
<p>A managed service that solves identity and data management; availability, synchronization, data security, and enables edge cloud computing specialized for the demanding requirements of a decentralized connected world.</p>
</div>
<div class="poin-item">
<div class="poin-item__img"><img alt="" class="base" src="assets/landing-poin2.svg"/><img alt="" class="white" src="assets/landing-poin2-white.svg"/></div>
<h2>Edge Computing Applications</h2>
<p>Our no-code data layer APIs are not only meant to be used by containers and server code, they are safe to be directly called from mobile devices, web user interfaces, or third party software (ecosystem).</p>
</div>
<div class="poin-item">
<div class="poin-item__img"><img alt="" class="base" src="assets/landing-poin3.svg"/><img alt="" class="white" src="assets/landing-poin3-white.svg"/></div>
<h2>SDK And Self-Service Portal</h2>
<p>Our low-code SDK is designed to allow customer onboarding through a policy governed self-service portal while ensuring seamless network discovery of the distributed edge data.</p>
</div>

<div class="intro-head">
<h1 class="line-big">Introduction</h1>
<p>CloudBackend takes its users from centralized to decentralized data, and enables fast, scalable and secure software. 
             To developers it is a turnkey database backend and a tool that a allows them to rapidly build new systems.</p>
</div>
<div class="intro-slide">
<img alt="pin" class="intro-slide__img" src="assets/img/pin.png"/>
<div class="intro-slide__content">
<h2 class="line-small">Edge acceleration as a Service</h2>
<div class="intro-slide__main" id="slide-container-1">
<div class="intro-slide__item"> <span>1</span>
<p>Sign up using our self-onboard portal. 
                   In the web interface you will find all the documentation and examples needed to get started. </p>
</div>
<div class="intro-slide__item"> <span>2</span>
<p>Download our latest SDK. 
                   The SDK is available in popular languages like C++, Java and Javascript. 
                   Wherever you run it
                   —
                   CloudBackend will be operational. </p>
</div>
<div class="intro-slide__item"> <span>3</span>
<p>Build your solution. 
                   The frontend, that is. 
                   We have already taken care of all backend data management and transfer between cloud and application. 
                   This includes identity management and authentication. 
                   Never before has something so complex been so simple.</p>
</div>
<div class="intro-slide__item"> <span>4</span>
<p>Launch your service. 
                   Up until now everything has been free of charge, 
                   but as your system is launched and starts to generate revenue it is also time to choose one of our plans. </p>
</div>
<div class="intro-slide__item"> <span>5</span>
<p>Congratulations! 
                   You have now launched a globally available, edge-accelerated service! 
                   All you have to do from here is to lean back and set the policies to your liking. 
                   We will handle the rest. </p>
</div>
</div>
<div class="intro-slide__arrow"><img alt="" onclick="scrollSlide(1,false)" src="assets/arrow-left.svg"/><img alt="" onclick="scrollSlide(1,true)" src="assets/arrow-right.svg"/></div>
</div>
</div>
<div class="intro-plain">
<div class="intro-plain__content">
<h2 class="line-small">Simple development with endless scalability</h2>
<p>Our platform manages deployment, operations, and scaling of required resources as well as 
               handles everything from central coordination of data, issuing of identities, 
               and the migration of data between cloud and device-edge, using policies. </p>
<p>A developer will experience this as working on a local database
               —
               while the user will get an offline-first edge experience, wherever in the world they might be. </p>
</div>
<img alt="scalability" src="assets/img/Synced_State.png"/>
</div>
<div class="intro-slide">
<img alt="Singularity Database" class="intro-slide__img" src="assets/img/Singularity_Database_across_verticals.png"/>
<div class="intro-slide__content">
<h2 class="line-small">Industrializing and standardizing data across cloud and edge</h2>
<div class="intro-slide__main" id="slide-container-2">
<div class="intro-slide__item"> <span>1</span>
<p>Set or modify policies for edge data acceleration needed at any given time and pay only for what you use.</p>
</div>
<div class="intro-slide__item"> <span>2</span>
<p>No backend development (no-code) and zero API/network code in clients 
                   makes building applications and systems much cheaper and less risky.</p>
</div>
<div class="intro-slide__item"> <span>3</span>
<p>Enabling low latency requirement use cases and innovative service creation through a 
                   low-code cross-compiling SDK for apps, containers, and cloud.</p>
</div>
<div class="intro-slide__item"> <span>4</span>
<p>This means that when you master the Singularity Database for one application, 
                   it can just as easily be applied in any other sector. 
                   Congratulations, you are now a specialized in all different application markets.</p>
</div>
</div>

<div class="intro-plain">
<div class="intro-plain__content">
<h2 class="line-small">Edge Native as the new default</h2>
<p>CloudBackend handles all data management at multiple levels end-to-end, 
               from the application all the way up to the public cloud. 
               All applications built on CloudBackend will be edge native by default, 
               but as easy to build as a local application.</p>
</div>
