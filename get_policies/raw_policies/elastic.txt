Elasticsearch</h1></div><div class="jsx-496183914 description"><p>The European Union's (EU) General Data Protection Regulation (GDPR) is now in effect. Learn about security and deployment best practices to get your Elasticsearch data compliant with GDPR guidelines.
</p></div><div class="jsx-496183914 mkto-form-wrapper inline"><p class="jsx-496183914">Talk with an Elastic expert about getting GDPR-ready with the Elastic Stack.</p><div class="jsx-3812957129 m-t-standard" id="marketo-fe-form"><div class="jsx-3812957129"><p class="jsx-3812957129 debug">MarketoFEForm</p><form class="jsx-3812957129" id="mktoForm_3958"></form></div></div></div></div></div></div><div class="layout layout-05"></div></div></div></section><div class="layout layout-06"></div><div class="jsx-750211597 title-text p-b-base-loose"><div class="jsx-750211597 container"><div class="jsx-750211597 row"><div class="jsx-750211597 title-text-section col-md-8 offset-md-2 text-center"><div class="jsx-3758713709 title-wrapper"><a class="jsx-3758713709" id="get-your-elasticsearch-data-gdprcompliant"></a><h2 class="jsx-3758713709" tag="">Get your Elasticsearch data GDPR-compliant</h2></div><div class="jsx-750211597 title-text-desc"><p>Whether you are using Elasticsearch for application search or as a centralized logging platform, there is a strong chance that you are dealing with data that is classified as personal data by GDPR guidelines, with strict requirements on how it's secured and processed.
</p><p>Learn how you can use Elastic Stack features, from role-based access control to data encryption, to get your Elasticsearch data <a href="/pdf/white-paper-of-gdpr-compliance-with-elastic-and-the-elastic-stack.pdf">ready for GDPR</a>.
</p></div></div></div></div></div><div class="layout layout-00"></div><div class="jsx-4084327366 image-text-grid"><div class="jsx-4084327366 container"><div class="jsx-4084327366 card-deck image-text-grid-icon32-3x2"><div class="jsx-4084327366 card one-column text-left"><div class="jsx-1750273902 icon-general"><figure class="jsx-1750273902 text-left"><img alt="" class="jsx-1750273902 m-b-16 image-32-icon" src="https://static-www.elastic.co/v3/assets/bltefdd0b53724fa2ce/bltd1114e49a833b654/5c307c064657a057675ba7ca/icon-security-events.svg"/></figure></div><div class="jsx-1058330117 title-text-cta card-body"><div class="jsx-1058330117 description"><p><strong>Access controls:</strong> <a href="https://www.elastic.co/guide/en/x-pack/current/authorization.html">Implement role-based access control</a>, down to the field level, to ensure that only authorized persons can access GDPR Personal Data in your Elasticsearch cluster.
</p></div></div></div><div class="jsx-4084327366 card one-column text-left"><div class="jsx-1750273902 icon-general"><figure class="jsx-1750273902 text-left"><img alt="" class="jsx-1750273902 m-b-16 image-32-icon" src="https://static-www.elastic.co/v3/assets/bltefdd0b53724fa2ce/bltad08214c8477017e/5c2004bb4cca137b38743534/icon-security-analytics.svg"/></figure></div><div class="jsx-1058330117 title-text-cta card-body"><div class="jsx-1058330117 description"><p><strong>Monitor access and breaches: </strong> Combine Elasticsearch <a href="https://www.elastic.co/guide/en/x-pack/current/auditing.html">audit and access logs</a> with machine learning and alerting jobs to get proactive with access monitoring and breach detection.
</p></div></div></div><div class="jsx-4084327366 card one-column text-left"><div class="jsx-1750273902 icon-general"><figure class="jsx-1750273902 text-left"><img alt="" class="jsx-1750273902 m-b-16 image-32-icon" src="https://static-www.elastic.co/v3/assets/bltefdd0b53724fa2ce/blt494772c9b541f6e6/5c3069c9ca40458566ef6761/icon-gdpr-pseudonymization.svg"/></figure></div><div class="jsx-1058330117 title-text-cta card-body"><div class="jsx-1058330117 description"><p><strong>Pseudonymization:</strong> Use Logstash fingerprint filter to <a href="/blog/gdpr-personal-data-pseudonymization-part-1">replace personal data with hashed values</a>.
</p></div></div></div><div class="jsx-4084327366 card one-column text-left"><div class="jsx-1750273902 icon-general"><figure class="jsx-1750273902 text-left"><img alt="" class="jsx-1750273902 m-b-16 image-32-icon" src="https://static-www.elastic.co/v3/assets/bltefdd0b53724fa2ce/blta873e28ea4981185/5c3069cdb7017c456eff0a29/icon-gdpr-encryption.svg"/></figure></div><div class="jsx-1058330117 title-text-cta card-body"><div class="jsx-1058330117 description"><p><strong>Encryption:</strong> <a href="https://www.elastic.co/guide/en/x-pack/6.2/encrypting-communications.html">Enable TLS / SSL to secure your data</a> in transit from snooping and tampering.
</p></div></div></div><div class="jsx-4084327366 card one-column text-left"><div class="jsx-1750273902 icon-general"><figure class="jsx-1750273902 text-left"><img alt="" class="jsx-1750273902 m-b-16 image-32-icon" src="https://static-www.elastic.co/v3/assets/bltefdd0b53724fa2ce/blt85e1cf907e61b725/5c3069c6c0508d076e171397/icon-gdpr-resilience-disaster-recovery.svg"/></figure></div><div class="jsx-1058330117 title-text-cta card-body"><div class="jsx-1058330117 description"><p><strong>Resilience and disaster recovery:</strong> Guard against loss of data with default index replication and horizontally scalable clustering.
</p></div></div></div></div></div></div><div class="layout layout-07"></div><div class="jsx-750211597 title-text p-b-base-loose"><div class="jsx-750211597 container"><div class="jsx-750211597 row"><div class="jsx-750211597 title-text-section col-md-8 offset-md-2 text-center"><div class="jsx-3758713709 title-wrapper"><a class="jsx-3758713709" id="use-the-elastic-stack-to-get-gdprcompliant"></a><h2 class="jsx-3758713709" tag="">Use the Elastic Stack to get GDPR-compliant</h2></div><div class="jsx-750211597 title-text-desc"><p>GDPR mandates that you "shall implement appropriate technical and organizational measures to ensure a level of security appropriate to the risk." </p><p>Many organizations are using the Elastic Stack as their central security analytics platform for real-time analysis of security information at scale.
